# Barter Code of Conduct

Barter is a small experimental project, but contributors and maintainers should still keep the repository respectful, focused, and safe to collaborate in.

## Expected Behavior

- Be respectful when discussing product direction, code changes, documentation, and design tradeoffs.
- Give feedback that is specific, actionable, and tied to the project.
- Assume changes should improve the current Flutter app or clarify the product requirements.
- Respect different experience levels with Flutter, Dart, mobile development, and product design.
- Keep discussions professional and avoid personal attacks.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private information without explicit permission.
- Repeated off-topic arguments that prevent project work from moving forward.
- Introducing malicious code, secrets, tracking, or unsafe dependencies.
- Misrepresenting planned Barter features as implemented production behavior.
